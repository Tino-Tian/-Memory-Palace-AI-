# 记忆宫殿 (Memory Palace)

给 Claude Code 装的长期记忆系统。每次新会话自动加载最近的记忆，聊天中主动嗅探相关内容，结束时自动归档——让你的 AI 助手不再失忆。

## 它解决了什么问题

默认的 Claude Code 每次新会话都像第一次见你，完全不记得之前聊过什么。记忆宫殿在背后维护一个 SQLite 数据库，把每段对话中有价值的内容存下来，下次对话时自动加载，让 AI 拥有"记得你"的能力。

## 安装

1. 把整个 `memory-palace` 目录放到你的用户目录下（`~/memory-palace/`）
2. 运行初始化脚本创建数据库：

```bash
sqlite3 ~/memory-palace/palace.db < ~/memory-palace/init.sql
```

3. 把启动钩子注册到 Claude Code，在 `~/.claude/settings.json` 的 hooks 里加上：

```json
"SessionStart": [
  {
    "type": "command",
    "command": "bash ~/memory-palace/hooks/startup.sh"
  }
]
```

搞定。下次新开会话时，记忆宫殿就会自动加载。

## 使用方法

不需要手动操作。AI 会自动在三种时机触发：

- **启动时** — 加载最近 3 天的记忆索引，注入到当前会话
- **聊天中** — 当话题触及过往记忆时，自动检索相关内容
- **结束时** — 把本次对话按话题拆分、归类、存入数据库

### 手动命令

如果你需要手动管理记忆，可以在聊天中直接使用 `/宫殿` 命令：

| 命令 | 作用 |
|------|------|
| `/宫殿 查找 <关键词>` | 搜索相关的历史记忆 |
| `/宫殿 类目 <路径>` | 查看某个类目下的所有记忆 |
| `/宫殿 树` | 查看完整的类目树 |
| `/宫殿 状态` | 查看数据库统计信息 |
| `/宫殿 归类 <路径>` | 把当前对话手动归入指定类目 |
| `/宫殿 新建类目 <路径>` | 手动创建一个新类目 |

## 目录结构

```
memory-palace/
├── README.md           # 本文件
├── LICENSE             # MIT 许可证
├── .gitignore          # 防止个人数据泄露
├── init.sql            # 数据库初始化脚本（建表 + 预设类目）
├── SKILL.md            # Claude Code 技能定义文件
├── hooks/
│   └── startup.sh      # 会话启动钩子（索引加载 + 衰减清理）
└── 归档/               # 超过 7 天的记忆自动打包压缩到这里
```

## 记忆的生命周期

```
raw (0-3天)
  ↓ 超过 3 天，自动浓缩
digest (3-7天)
  ↓ 超过 7 天，打包压缩到 归档/
归档文件 (永久保留，可手动搜索)
```

每条记忆只保留结论和关键上下文，对话中的闲聊和来回讨论会自动丢弃。

## 隐私注意

- 所有数据存在本地 `palace.db`，不上传任何地方
- 归档文件用 gzip 压缩存放在本地
- 发布到 GitHub 时，`.gitignore` 已配置忽略 `palace.db` 和归档文件
