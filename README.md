# 记忆宫殿 Memory Palace

AI 助手的长期记忆系统。基于 SQLite，支持智能语义归类、时间衰减、归档压缩。

## 核心特性

- **递归智能归类** — AI 读懂每段对话，从根类目逐层往下匹配。现有类目装不下就自动创建子类。不是关键词匹配，是真正的语义分析。
- **类目自然生长** — 预设类目只是起点。你是设计师就长出设计树，程序员长出开发树，金融从业者长出投资树。系统通过聊天来认识你。
- **时间衰减** — 0-3 天保留完整摘要（raw），3-7 天浓缩为核心（digest），7 天后压缩归档。模拟人脑遗忘曲线。
- **自动嗅探** — 聊到相关话题时，AI 自动从记忆库检索历史记录，不用重复解释背景。
- **纯本地运行** — SQLite 存于本地，不联网、不上传、不依赖云服务。

## 真实数据：一年下来发生了什么

假设每天 3-5 段有价值对话，每段一条记忆摘要（约 200 字）。

### 磁盘占用（"伤害"）

| 阶段 | 状态 | 占用 |
|------|------|------|
| 每天新增 | 3-5 条 raw 记录 | ~2 KB |
| 3 天后 | raw → digest 浓缩 | 缩小约 60% |
| 7 天后 | .gz 压缩归档 | 再压缩约 80% |
| **一年累计** | | **约 1-2 MB** |

一年只多 1-2 MB。一张手机照片 5-10 MB，微信一天缓存 200-500 MB。记忆宫殿一年的"伤害"不到半张照片大小。

### 短期增益

- **启动加载** — 每次新会话只加载索引（~50 token），相当于只看文件夹名。
- **自动嗅探** — 聊到相关话题 AI 自动翻历史记忆，不用重新解释背景。
- **自动归档** — 对话结束前 AI 自己按话题分类存好，无需任何操作。

### Token 成本

| 项目 | 每次 | 频率 | 一年 |
|------|------|------|------|
| 启动索引加载 | ~50 token | 每天 3 次 | ~5.5 万 |
| 对话中嗅探 | ~30 token | 每天 2 次 | ~2.2 万 |
| 归档写入 | ~20 token | 每天 3 次 | ~2.2 万 |
| **一年总计** | | | **约 10 万 token** |

每次重述背景花 200-500 token，一天一次就是 7-18 万/年。**记忆宫殿省下的远超过自身消耗。** 按 Claude API 价格（Sonnet $3/$15 per 1M tokens），一年成本约 $0.30-$1.50。

## 分类体系：以你为中心

系统预设了「工作」和「生活」两个根类目及常用子类目，但**这些只是起点，不是模板**。分类体系以你的实际工作为中心自然生长。

### 预设只是起点

```
工作                                生活
├── 设计                            ├── 国际大事
│   ├── UI设计                      │   ├── 国家负责人讲话
│   └── 平面设计                    │   ├── 国际冲突
├── 运营                            │   └── 外交动向
│   ├── 活动策划                    ├── 科技/AI资讯
│   └── 内容运营                    ├── 行政动态
├── 剪辑                            ├── 财经
├── 短视频                          └── 健康
│   ├── 脚本
│   └── 拍摄
└── 开发
    ├── 前端
    └── 后端
```

### 当预设完全不适用时

假设你是一个金融从业者，第一次聊工作：

> 用户："今天分析了茅台的 K 线形态，准备在美联储决议前减仓"

AI 的归类过程：

1. 根层级：他在做投资决策 → 属于「工作」
2. 进入「工作」：看到 设计、运营、剪辑、短视频、开发
3. 逐一判断：「股票投资」不属于任何一个 → 「工作」是分叉点
4. 「股票投资」还能细分 → K线、美联储、减仓……多个维度
5. 在「工作」下创建「股票投资」中类
6. 继续递归：K线形态 → 「技术分析」小类；美联储决议 → 「宏观研究」小类
7. 归入：`工作/股票投资/技术分析`

第二天聊"用 DCF 模型给茅台估值"：

> AI：工作 → 股票投资 → 技术分析不匹配 → 创建「估值分析」小类 → 归入 `工作/股票投资/估值分析`

一周后，你的类目树：

```
工作
└── 股票投资        ← 自动创建
    ├── 技术分析     ← 自动创建
    ├── 宏观研究     ← 自动创建
    └── 估值分析     ← 自动创建
```

预设的那些设计、运营类目你从未聊过，它们静静待着，不占资源、不影响归类。**系统不会预设你是什么职业——它通过跟你聊天来认识你。**

## 安装（一行命令）

### macOS / Linux

```bash
bash install.sh
```

或通过 curl：

```bash
curl -fsSL https://raw.githubusercontent.com/Tino-Tian/-Memory-Palace-AI-/main/install.sh | bash
```

自动完成：检测系统 → 安装 sqlite3 → 创建目录 → 写入文件 → 初始化数据库 → 配置 Claude Code。

### Windows

```powershell
.\install.ps1
```

或一行：

```powershell
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/Tino-Tian/-Memory-Palace-AI-/main/install.ps1" -OutFile "install.ps1"; .\install.ps1
```

自动完成：检测 sqlite3 → winget 安装 → 创建目录 → 写入文件 → 初始化数据库。

> 装完即用。Claude Code 下次启动自动加载；其他 AI 平台粘贴 SKILL.md 到系统提示词即可。

## 使用方法

### 自动模式（无需操作）

| 时机 | 行为 |
|------|------|
| 启动时 | 加载最近 3 天记忆索引 |
| 聊天中 | 自动检索相关历史记忆 |
| 结束时 | 拆分话题、智能归类、存入数据库 |

### 手动命令

| 命令 | 作用 |
|------|------|
| `/宫殿 查找 <关键词>` | 搜索历史记忆（含归档） |
| `/宫殿 类目 <路径>` | 查看指定类目下的记忆 |
| `/宫殿 树` | 查看完整类目树 |
| `/宫殿 状态` | 查看数据库统计 |
| `/宫殿 归类 <路径>` | 手动归入指定类目 |
| `/宫殿 新建类目 <路径>` | 手动创建新类目 |

## 记忆的生命周期

```
raw (0-3天)
  ↓ 超过 3 天，自动浓缩
digest (3-7天)
  ↓ 超过 7 天，打包压缩到 归档/
归档文件 (永久保留，可手动搜索)
```

每条记忆只保留结论和关键上下文，闲聊和回顾过程自动丢弃。
