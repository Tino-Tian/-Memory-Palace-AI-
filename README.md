# 记忆宫殿 (Memory Palace)

AI 助手的长期记忆系统。基于 SQLite，支持自动语义归类、时间衰减、归档压缩。适配 Claude Code、ChatGPT、Cursor 等所有主流 AI 平台。

## 它解决了什么问题

AI 助手默认每次新会话都像第一次见你，完全不记得之前聊过什么。记忆宫殿在后台维护一个 SQLite 数据库，把每段对话中有价值的内容存下来，下次对话时自动加载。

不只是存——它还会**智能归类**。AI 会读懂你的每段对话内容，然后像人整理文件一样：判断该放入哪个现有分类、还是需要新建一个文件夹。这个分类体系会随着你的使用自然生长。

## 真实数据：一年下来发生了什么

我们用高频使用场景来算一笔账。假设你每天跟 AI 进行 3-5 段有价值的对话（闲聊不算），每段产生一条记忆摘要（约 200 字）。

### 对系统的"伤害"（磁盘占用）

| 时间 | 状态 | 占用空间 |
|------|------|----------|
| 每天新增 | 3-5 条 raw 记录 | ~2 KB |
| 3 天后 | raw → digest 浓缩 | 缩小约 60% |
| 7 天后 | 打包归档为 .gz 压缩文件 | 再压缩约 80% |
| **一年下来** | **总计** | **约 1-2 MB** |

一年高频使用，在你电脑上只多了一个 1-2 MB 的文件夹。作为对比：一张手机照片 5-10 MB，微信一天的缓存 200-500 MB。记忆宫殿一年的"伤害"还不到半张照片的大小。

### 短期增益

- **启动加载** — 每次新会话只加载索引（~50 token），不看完整内容。相当于只看文件夹名，不打开每个文件。
- **自动嗅探** — 聊到相关话题时，AI 自动翻出对应的历史记忆，不用你重新解释背景。比如你上周讨论过"特朗普访华"，今天提到相关话题，AI 自动调出那条记忆。
- **自动归档** — 对话结束前，AI 自己把本次内容按话题分类存好。你不需要做任何操作。

### Token 成本

| 项目 | 每次消耗 | 假设频率 | 一年总量 |
|------|----------|----------|----------|
| 启动索引加载 | ~50 token | 每天 3 次启动 | ~5.5 万 token |
| 对话中嗅探 | ~30 token | 每天 2 次命中 | ~2.2 万 token |
| 结束时归档写入 | ~20 token | 每天 3 次对话 | ~2.2 万 token |
| **一年总计** | | | **约 10 万 token** |

10 万 token 看起来不少，但换个角度：如果没有记忆系统，你每次跟 AI 重述背景就要花 200-500 token。假设每天只有 1 次需要重述背景，一年就是 7-18 万 token。**记忆宫殿帮你省掉的 token 远超过它自身消耗的。**

按 Claude 的 API 价格（Sonnet $3/$15 per 1M token），一年 10 万 token 的成本大约是 $0.30-$1.50。省下的重述成本是这个数字的 2-3 倍。

## 核心特性

- **递归智能归类** — AI 从根类目开始逐层往下匹配，找到最合适的层级。现有类目装不下就自动创建子类，分类体系自然生长
- **时间衰减** — 0-3 天保留完整摘要，3-7 天浓缩为核心要点，7 天后打包归档。模拟人脑的遗忘曲线
- **对话中嗅探** — 当你聊到相关话题时，AI 自动从记忆库中检索历史记录来辅助当前对话
- **纯本地运行** — 所有数据存在你电脑上的 SQLite 文件里，不联网、不上传、不依赖任何云服务

## 适配平台

记忆宫殿的核心是一套通用的"系统提示词 + SQL 脚本"，不绑定任何特定平台。

| 平台 | 接入方式 |
|------|----------|
| Claude Code | 将 SKILL.md 放入技能目录，配置 SessionStart 钩子 |
| ChatGPT / 网页版 | 将 SKILL.md 全文粘贴到"自定义指令"中 |
| Cursor / 其他 IDE | 将 SKILL.md 加入项目规则文件（.cursorrules 等） |
| API 调用 | 将 SKILL.md 写入 system prompt |
| 任何支持自定义指令的 AI | 同上，粘贴指令即可使用 |

不同平台的区别仅在于"初始化数据库"和"启动时加载索引"的实现方式（脚本 vs 手动 vs 自动）。核心的归类逻辑、衰减机制、命令体系完全通用。

## 安装（一行命令）

### macOS / Linux

打开终端，粘贴以下命令并回车：

```bash
bash install.sh
```

或者如果你已经有 GitHub 仓库的 URL：

```bash
curl -fsSL https://raw.githubusercontent.com/<你的用户名>/memory-palace/main/install.sh | bash
```

脚本会自动完成：检测系统 → 安装 sqlite3（如需要）→ 创建目录 → 写入所有文件 → 初始化数据库 → 配置 Claude Code。

### Windows

在 PowerShell 中运行（右键 → 使用 PowerShell 运行 `install.ps1`）：

```powershell
.\install.ps1
```

或一行命令：

```powershell
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/<你的用户名>/memory-palace/main/install.ps1" -OutFile "install.ps1"; .\install.ps1
```

脚本会自动完成：检测 sqlite3 → 通过 winget 自动安装（如需要）→ 创建目录 → 写入所有文件 → 初始化数据库。

> 装完之后就没事了。下次启动 Claude Code 时记忆宫殿会自动加载。其他 AI 平台只需将 SKILL.md 的内容粘贴到系统提示词中。

## 使用方法

### 自动模式（无需操作）

AI 会在三个时机自动触发：

- **启动时** — 加载最近 3 天的记忆索引，注入到当前会话
- **聊天中** — 当话题触及过往记忆时，自动检索相关历史记录
- **结束时** — 把本次对话按话题拆分、智能归类、存入数据库，处理过期记忆

### 手动命令

在聊天中直接输入（不同平台的命令前缀可能不同，以下以 Claude Code 为例）：

| 命令 | 作用 |
|------|------|
| `/宫殿 查找 <关键词>` | 搜索历史记忆（含归档） |
| `/宫殿 类目 <路径>` | 查看指定类目下的所有记忆 |
| `/宫殿 树` | 查看完整类目树 |
| `/宫殿 状态` | 查看数据库统计信息 |
| `/宫殿 归类 <路径>` | 手动归入指定类目（覆盖自动归类） |
| `/宫殿 新建类目 <路径>` | 手动创建新类目 |

## 分类体系

系统预设了"工作"和"生活"两个根类目，下面是围绕常见职业预设的一些子类目（设计、运营、短视频、开发等）。

**但这些只是起点，不是模板。**

分类体系会随着你的对话内容动态生长。更重要的是——它是**以你的实际工作内容为中心**来生长的，不会硬把你往预设类目里塞。以下是一个完整的跨职业例子：

### 例子：预设类目完全不适配时

假设你是一个金融从业者，第一次跟 AI 聊工作：

```
用户："今天分析了茅台的 K 线形态，准备在美联储决议前减仓"

AI 的归类过程：
1. 根层级：属于"工作"还是"生活"？→ 他在做投资决策，这是工作
2. 进入"工作"：看到 设计、运营、剪辑、短视频、开发
3. 逐一判断："股票投资"不属于任何一个 → "工作"是分叉点
4. "股票投资"这个主题还能细分吗？→ K线、美联储、减仓……涉及多个维度
5. 在"工作"下创建"股票投资"中类
6. 进入"股票投资"，继续递归：
   - "K线形态" → 创建"技术分析"小类
   - "美联储决议" → 创建"宏观研究"小类
7. 内容归入：工作/股票投资/技术分析
```

第二天聊"用 DCF 模型给茅台估值"：

```
AI：工作 → 股票投资 → 技术分析不匹配 → 创建"估值分析"小类
内容归入：工作/股票投资/估值分析
```

一周后，你的类目树变成：

```
工作
└── 股票投资        ← AI 自动创建的
    ├── 技术分析     ← AI 自动创建的
    ├── 宏观研究     ← AI 自动创建的
    └── 估值分析     ← AI 自动创建的
```

预设的那些"设计、运营、短视频"类目你从未聊过，它们就静静待在那里，不占资源、不影响归类。**系统不会预设你是什么职业——它通过跟你聊天来"认识"你。**

你是设计师，它就长出设计类目树；你是程序员，它就长出开发类目树；你是金融从业者，它就长出投资类目树。同一套系统，不同用户安装后会长出完全不同的"大脑"。

## 记忆的生命周期

```
raw (0-3天)
  ↓ 超过 3 天，自动浓缩
digest (3-7天)
  ↓ 超过 7 天，打包压缩到 归档/
归档文件 (永久保留，可手动搜索)
```

每条记忆只保留结论和关键上下文，闲聊和来回讨论自动丢弃。

## 目录结构

```
memory-palace/
├── README.md           # 本文件
├── LICENSE             # MIT 许可证
├── .gitignore          # 防止个人数据泄露
├── init.sql            # 数据库初始化（建表 + 预设类目）
├── SKILL.md            # 核心指令（AI 行为定义，所有平台通用）
├── hooks/
│   └── startup.sh      # Claude Code 启动钩子（其他平台可忽略）
└── 归档/               # 超过 7 天的记忆打包压缩存放
```

## 隐私

- 所有数据存于本地 `palace.db`，不上传任何服务器
- `.gitignore` 已配置忽略数据库和归档文件，发布代码不会泄露个人数据
- 随时可删除 `palace.db` 和 `归档/` 来清空所有记忆
